# 중소기업수출지원센터 (smesgo) 스크래퍼 개발 인사이트

## 개발 일시: 2025-07-16
## 대상 사이트: https://www.smes.go.kr/exportcenter/information/notice/list.do

## 1. 사이트 특징 및 구조 분석

### 1.1 사이트 구조
- 표준적인 정부기관 웹사이트 구조
- 공지사항은 테이블 형태로 목록 표시
- 페이지네이션: 하단 숫자 페이지 링크
- 상세 페이지: 제목, 내용, 첨부파일 구조

### 1.2 HTML 구조
- 목록 페이지: `table.table-type-list > tbody > tr`
- 상세 페이지: `div.post-content`, `ul.post-file-list`
- 첨부파일: `/exportcenter/information/notice-files/download.do` 경로

### 1.3 데이터 특성
- 총 2,249개의 공지사항 등록
- 주요 내용: 수출 지원 정책, 관세 정보, 사업 공고
- 첨부파일: 대부분 hwp, pdf, xlsx 형태

## 2. 스크래핑 과정에서 발견한 기술적 특징

### 2.1 세션 관리
- CSRF 토큰 사용 (`_csrf` 메타태그)
- 세션 유지 필요 (첨부파일 다운로드 시)

### 2.2 첨부파일 다운로드
- 일부 파일 다운로드 실패 (0 바이트 파일 생성)
- 실패 원인: 링크 만료 또는 접근 권한 문제
- 파일 크기 검증 필요

### 2.3 한글 파일명 처리
- UTF-8 인코딩 정상 지원
- 특수문자 처리 필요 (파일시스템 호환성)

## 3. 성능 및 안정성

### 3.1 요청 속도
- 페이지 간 3초 간격 적용
- 공고 간 2초 간격 적용
- 첨부파일 다운로드 간 1초 간격

### 3.2 오류 처리
- 네트워크 타임아웃: 30초 (목록), 60초 (첨부파일)
- HTML 응답 감지 및 자동 제거
- 작은 파일 크기 검증 (1KB 미만)

## 4. 수집 결과 분석

### 4.1 테스트 결과 (3페이지)
- 총 처리 공고: 25개
- 다운로드 첨부파일: 40개 (0 바이트 파일 3개 제외)
- 한글 파일명: 완전 지원
- 파일 크기 중복: 2개 (동일 파일로 정상)

### 4.2 콘텐츠 품질
- 공고 본문: 이미지 포함 HTML 형태
- 첨부파일: 공식 문서 형태
- 메타데이터: 작성일, 조회수, URL 정보

## 5. 개선 사항 및 권장사항

### 5.1 기술적 개선
- 첨부파일 다운로드 재시도 메커니즘
- 병렬 처리 도입 (주의: 서버 부하 고려)
- 중복 파일 감지 및 스킵 기능

### 5.2 운영 권장사항
- 주기적 실행 시 증분 수집 고려
- 파일 크기 모니터링 및 알림
- 다운로드 실패 파일 별도 관리

## 6. 사이트별 특수 고려사항

### 6.1 정부기관 특성
- 공식 문서 중심의 콘텐츠
- 정책 변경에 따른 긴급 공지 빈발
- 첨부파일 중요도 높음

### 6.2 접근성 고려사항
- robots.txt 무시 (지침에 따라)
- 적절한 User-Agent 사용
- 서버 부하 최소화 노력

## 7. 스크래퍼 코드 핵심 특징

### 7.1 모듈화 설계
- 클래스 기반 구조
- 메서드별 기능 분리
- 설정 가능한 매개변수

### 7.2 오류 처리
- 로깅 시스템 활용
- 예외 상황 대응
- 검증 메커니즘 내장

### 7.3 출력 형태
- 디렉토리 구조: output/site_code/번호_제목/
- 콘텐츠: content.md (마크다운)
- 첨부파일: attachments/ 폴더

## 8. 결론

중소기업수출지원센터 사이트는 표준적인 정부기관 웹사이트 구조를 갖추고 있어 
스크래핑이 비교적 안정적으로 가능합니다. 다만 일부 첨부파일의 다운로드 실패와 
서버 응답 시간 등을 고려한 안정성 확보가 중요합니다.
